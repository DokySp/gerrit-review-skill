# Gerrit Review 스킬

Gerrit patch set 리뷰를 조회하고, AI reviewer 코멘트의 유효성을 실제 코드 기준으로 검토한 뒤, Gerrit 답변 초안이나 draft comment, 실제 review comment를 준비하는 Codex 스킬입니다.

기본 동작은 읽기 전용 분석과 dry-run입니다. Gerrit에 draft나 실제 코멘트를 생성하려면 사용자가 명시적으로 요청해야 합니다.

스킬을 사용하려면 먼저 Gerrit 계정 설정이 필요합니다. 설정이 없으면 스크립트는 Gerrit 조회, draft 생성, 실제 게시를 수행하지 않고 설정 안내만 출력합니다.

## 설치 위치

이 저장소의 실제 스킬 폴더는 `skills/gerrit-review`입니다. 사용하는 환경에 맞게 이 폴더를 다음 위치 중 하나에 배치합니다.

```bash
# 프로젝트별로 함께 배포할 때
mkdir -p .agents/skills
cp -R skills/gerrit-review .agents/skills/gerrit-review

# 개인 전역 스킬로 사용할 때
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R skills/gerrit-review "${CODEX_HOME:-$HOME/.codex}/skills/gerrit-review"
```

두 설치 위치 모두 지원합니다. 스크립트는 설치 루트가 아니라 스킬 폴더 자체를 기준으로 `config/account.json`, `references/`, `scripts/`를 찾습니다. 터미널에서 직접 스크립트를 실행할 때는 설치된 `gerrit-review` 폴더에서 실행하거나 해당 폴더의 `scripts/...` 경로를 명시합니다.

## 주요 기능

- 현재 로컬 git checkout의 `HEAD`, `Change-Id`, remote, branch 정보를 수집합니다.
- 로컬 Gerrit 계정 설정을 생성하고 확인합니다.
- Gerrit SSH 연결 상태를 진단합니다.
- Gerrit change와 patch set 정보를 조회합니다.
- AI reviewer의 inline comment를 파일/라인 단위로 정리합니다.
- 코멘트 대상 파일과 호출 경로를 실제 코드에서 확인합니다.
- 각 코멘트를 `valid`, `partially_valid`, `false_positive`, `needs_more_context`로 분류합니다.
- `false_positive`로 판단된 코멘트에 대해 Gerrit 답변 초안을 만듭니다.
- Codex가 작성하는 모든 Gerrit comment 앞에 `[Codex로 생성됨]` prefix를 붙입니다.
- 실제 review comment용 `ReviewInput` JSON을 생성합니다.
- REST 인증이 가능한 경우 draft comment 요청을 dry-run 또는 실제 실행으로 생성합니다.

## 기본 워크플로

1. Gerrit 계정 설정이 있는지 확인합니다.
2. 설정이 없으면 작업을 중단하고 설정 명령을 안내합니다.
3. 로컬 기준 정보를 수집합니다.
4. Gerrit SSH 접근 가능 여부를 확인합니다.
5. 현재 `HEAD`, `Change-Id`, 또는 사용자가 입력한 change number로 Gerrit 리뷰를 조회합니다.
6. Gerrit current patch set revision과 로컬 `HEAD`가 일치하는지 확인합니다.
7. AI reviewer inline comment를 추출합니다.
8. 각 코멘트를 실제 코드와 호출 경로 기준으로 검토합니다.
9. Gerrit에 남길 답변 초안을 작성합니다.
10. 기본적으로 dry-run JSON만 생성합니다.
11. 사용자가 명시적으로 요청한 경우에만 draft를 생성합니다.
12. draft 생성 후에는 작업을 종료합니다. `진행해`, `올려`, `게릿에 올려` 같은 후속 표현만으로 draft를 publish하지 않습니다.
13. 실제 게시나 draft publish는 별도 명시 요청과 2차 확인이 있을 때만 수행합니다.

## 계정 설정

기본 설정 파일 위치는 스킬 내부의 별도 파일입니다.

```text
config/account.json
```

배포본에는 빈 `config/account.json`을 포함합니다. 이 파일에는 실제 사용자, password, base URL 값을 넣지 않은 상태로 배포해야 합니다.

Gerrit 계정과 REST draft 생성에 필요한 값은 스킬 내부 설정 파일의 `account.user`, `account.password`, `account.base_url`에 저장합니다. `account.user`는 SSH와 REST에 공통으로 사용하며, 별도의 `ssh.user`나 `rest.user`는 저장하지 않습니다.

SSH `host`와 `port`, Gerrit project/package는 설정 파일에 저장하지 않습니다. Gerrit 조회 시 로컬 repo의 `git remote -v`에서 추론하며, 특수한 경우에만 `scripts/gerrit_query.py --host <host> --port <port>`로 override합니다.

`account.base_url`의 추가 경로는 Gerrit REST context path입니다. project/package 설정값과 다르므로, 해당 Gerrit 서버가 그 경로에서 REST를 제공하면 유지해야 합니다. 이 값은 일반 출력에서 redaction 처리합니다.

```json
{
  "account": {
    "user": "<gerrit-user>",
    "password": "<HTTP_PASSWORD>",
    "base_url": "<gerrit-rest-base-url>"
  }
}
```

기본적으로 이 파일만 사용합니다. 다른 파일을 테스트해야 할 때만 `--config` 인자로 명시합니다.

```bash
scripts/configure_account.py --config /path/to/account.json --show
```

계정 설정을 대화형으로 생성하거나 갱신합니다.

```bash
scripts/configure_account.py
```

Gerrit 사용자, REST base URL, password 또는 auth header는 프롬프트에만 입력합니다. 계정 값, password, base URL, auth header를 shell command 인자로 넣지 않습니다.

이미 설정된 값이 있으면 같은 명령으로 필요한 값만 갱신할 수 있습니다.

```bash
scripts/configure_account.py
```

현재 설정을 확인합니다.

```bash
scripts/configure_account.py --show
```

## 스크립트

### `scripts/gerrit_query.py`

Gerrit SSH query를 실행하고 결과를 정규화합니다.

```bash
scripts/configure_account.py --show
scripts/gerrit_query.py --diagnose
scripts/gerrit_query.py --change 1837786
scripts/gerrit_query.py --query change:I35167bb952735969c2ac4ab1e55044e80649c580
```

출력에는 로컬 git 정보, Gerrit change 정보, current patch set, approval, reviewer, change-level message, inline comment, `HEAD` 정합성 결과가 포함됩니다.

### `scripts/build_review_input.py`

답변 초안 JSON을 Gerrit `ReviewInput` JSON으로 변환합니다.
모든 comment message와 change-level review message 앞에는 `[Codex로 생성됨]`이 자동으로 붙습니다.

```bash
scripts/build_review_input.py replies.json --message "Review analysis dry-run"
```

기본값은 다음과 같습니다.

- `notify: NONE`
- `omit_duplicate_comments: true`
- `tag: autogenerated:codex-gerrit-review`

### `scripts/create_draft_comments.py`

REST draft comment 요청을 생성합니다. 기본값은 dry-run이며, 실제 생성은 `--execute`가 있을 때만 수행합니다.
모든 draft comment message 앞에는 `[Codex로 생성됨]`이 자동으로 붙습니다.

```bash
scripts/create_draft_comments.py replies.json \
  --change 12345 \
  --revision 3
```

실제 draft 생성을 하려면 `config/account.json`의 REST 인증 필드와 `--execute`가 필요합니다.

### `scripts/publish_drafts.py`

이미 생성된 draft를 실제 comment로 publish합니다. 기본값은 dry-run입니다.

```bash
scripts/publish_drafts.py --change 12345 --revision 3
scripts/publish_drafts.py --change 12345 --revision 3 \
  --execute --confirm PUBLISH_DRAFTS:12345,3
```

이 스크립트는 사용자가 명시적으로 `draft를 실제 게시해줘`라고 요청하고, 대상 확인 후 2차 확인이 있을 때만 사용합니다.

## 안전 규칙

- 사용자가 명시적으로 요청하지 않으면 Gerrit에 아무것도 게시하지 않습니다.
- Gerrit 계정 설정이 없으면 추가 작업을 수행하지 않고 설정 안내만 출력합니다.
- 기본 출력은 분석 결과와 dry-run JSON입니다.
- draft 생성은 실제 게시가 아닙니다. draft 요청 후에는 draft 생성에서 멈춥니다.
- `진행해`, `올려`, `게릿에 올려` 같은 표현은 실제 게시 승인으로 해석하지 않습니다.
- 실제 게시 전에는 publish-specific 요청과 2차 확인이 모두 필요합니다.
- `submit`, `abandon`, `restore`, `rebase`, `move`, label vote는 이 스킬의 기본 기능에서 제외합니다.
- 실제 게시는 comment-only로 제한합니다.
- 로컬 `HEAD`와 Gerrit current patch set revision이 다르면 실제 게시를 금지합니다.
- `false_positive` 판정에는 반드시 실제 코드 파일/라인 근거가 필요합니다.
- 근거가 부족하면 `false_positive` 대신 `needs_more_context`로 분류합니다.

## 답변 형식

리뷰 분석 결과는 기본적으로 다음 순서로 정리합니다.

- 요약
- patch set 정보
- 코멘트별 판정
- false positive 목록
- valid issue 목록
- Gerrit reply 초안
- dry-run JSON
- 실제 게시 여부

이 사용자의 저장소에서는 최종 답변을 `사실`과 `추측`으로 나누어 작성합니다.
