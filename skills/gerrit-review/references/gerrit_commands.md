# Gerrit Commands

Before running Gerrit commands through this skill, verify account config with:

```bash
scripts/configure_account.py --show
```

If config is missing, stop and follow `references/account_config.md`.

## SSH Query

Use this command to fetch the current patch set, patch sets, approvals, comments, reviewers, and submit records:

```bash
ssh -o BatchMode=yes -o ConnectTimeout=10 -p <port> <user>@<host> \
  gerrit query --format=JSON --current-patch-set --patch-sets --all-approvals \
  --comments --all-reviewers --submit-records <query> limit:1
```

`gerrit query --format=JSON` returns JSON lines. Ignore the row with `"type":"stats"`.

Good query choices:

- Commit SHA: `c55e838323c1094e043b7649fea673290fdf2f01`
- Change-Id: `change:I35167bb952735969c2ac4ab1e55044e80649c580`
- Change number: `1837786`

## Actual Review Posting

Use this command only after explicit actual-comment publish intent and second confirmation:

```bash
ssh -p <port> <user>@<host> gerrit review --json <change>,<patchset>
```

Pass a Gerrit `ReviewInput` JSON object on stdin:

```json
{
  "message": "Gerrit review result.",
  "tag": "autogenerated:codex-gerrit-review",
  "notify": "NONE",
  "omit_duplicate_comments": true,
  "comments": {
    "path/to/File.kt": [
      {
        "line": 122,
        "message": "[Codex로 생성됨] 현재 호출 경로는 lifecycleScope.launch 기반이며 Main dispatcher에서만 실행됩니다.",
        "unresolved": false
      }
    ]
  }
}
```

Do not include `labels` unless the user explicitly asks for a vote.

Do not use this command after a draft-only request. If the user asked for drafts, the task ends after draft creation.

## REST Draft Comments

Use draft creation only when REST authentication is known to work for the Gerrit host.

Prefer `scripts/create_draft_comments.py` for dry-run request generation and optional creation.

Endpoint:

```text
PUT /changes/{change-id}/revisions/{revision-id}/drafts
```

Minimal `CommentInput`:

```json
{
  "path": "path/to/File.kt",
  "line": 122,
  "message": "[Codex로 생성됨] 현재 코드 기준으로 해당 호출 경로는 확인되지 않습니다.",
  "unresolved": false
}
```

Useful REST endpoints:

- `GET /changes/{change-id}/revisions/{revision-id}/drafts/`
- `PUT /changes/{change-id}/revisions/{revision-id}/drafts/{draft-id}`
- `DELETE /changes/{change-id}/revisions/{revision-id}/drafts/{draft-id}`

Drafts belong to the calling user and can be edited or deleted before publication.

## Publishing Existing Drafts

Publishing drafts is not the same operation as creating drafts. Do not infer publish permission from `진행해`, `올려`, `게릿에 올려`, `계속해`, or similar follow-ups after draft creation.

Use only after the user explicitly asks to publish existing drafts and then confirms the exact target:

```bash
scripts/publish_drafts.py --change <change> --revision <patchset>
scripts/publish_drafts.py --change <change> --revision <patchset> \
  --execute --confirm PUBLISH_DRAFTS:<change>,<patchset>
```

The script calls:

```text
POST /changes/{change-id}/revisions/{revision-id}/review
```

with:

```json
{
  "drafts": "PUBLISH",
  "notify": "NONE",
  "tag": "autogenerated:codex-gerrit-review"
}
```

Do not call this REST endpoint manually from the skill workflow.
